# MCP OAuth — Account Ownership Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Let MCP users authenticate the `/mcp` connection with Google or GitHub (via Supabase's OAuth 2.1 server) so their tool calls own their artifacts, while anonymous (edit-token) use keeps working unchanged.

**Architecture:** Supabase is the **Authorization Server** (OAuth server + DCR + JWT issuance + the Google/GitHub login on our consent page). Our `/mcp` is the **Resource Server**: mcp-handler's `withMcpAuth` validates the Supabase JWT via JWKS (`jose`), and the validated user id is threaded into the existing service as `owner_id`. No DB migration — `owner_id` already exists.

**Tech Stack:** Next.js 16 (App Router, Node runtime), `mcp-handler` (`withMcpAuth` / `protectedResourceHandler`), `@modelcontextprotocol/sdk`, `jose` (JWKS validation), `@supabase/supabase-js` (browser auth + OAuth consent API), Supabase OAuth 2.1 server (beta), Vitest 3.

**Spec:** `docs/superpowers/specs/2026-06-02-mcp-oauth-design.md`

- [ ] **Step 3: Implement `lib/mcp/auth.ts`**

```typescript
import { createRemoteJWKSet, jwtVerify, type JWTVerifyGetKey } from 'jose';
import type { AuthInfo } from '@modelcontextprotocol/sdk/server/auth/types.js';

export interface VerifyDeps {
  jwks: JWTVerifyGetKey;
  issuer: string;
}

/**
 * Build a verifyToken function for mcp-handler's withMcpAuth. Validates a Supabase
 * access-token JWT against the given JWKS + issuer. Returns AuthInfo on success, or
 * undefined for missing/invalid/expired/wrong-issuer tokens (the caller treats
 * undefined as anonymous — this endpoint is intentionally dual-mode).
 */
export function makeVerifyMcpToken({ jwks, issuer }: VerifyDeps) {
  return async function verifyMcpToken(
    _req: Request,
    bearerToken?: string,
  ): Promise<AuthInfo | undefined> {
    if (!bearerToken) return undefined;
    try {
      const { payload } = await jwtVerify(bearerToken, jwks, { issuer });
      const userId = typeof payload.sub === 'string' ? payload.sub : undefined;
      if (!userId) return undefined;
      return {
        token: bearerToken,
        clientId: typeof payload.client_id === 'string' ? payload.client_id : 'unknown',
        scopes: [],
        expiresAt: typeof payload.exp === 'number' ? payload.exp : undefined,
        extra: { userId },
      };
    } catch {
      return undefined; // fail closed to anonymous, never to a trusted identity
    }
  };
}

const SUPABASE_URL = process.env.NEXT_PUBLIC_SUPABASE_URL ?? '';
const ISSUER = `${SUPABASE_URL}/auth/v1`;
// Lazy remote JWKS (fetched on first verification, then cached by jose).
const remoteJwks = createRemoteJWKSet(new URL(`${SUPABASE_URL}/auth/v1/.well-known/jwks.json`));

export const verifyMcpToken = makeVerifyMcpToken({ jwks: remoteJwks, issuer: ISSUER });
```

> Note: the module-level `new URL(...)` requires `NEXT_PUBLIC_SUPABASE_URL` to be a valid URL. It is set in `.env.local` (loaded by `vitest.setup.ts`) and in Vercel. Tests import only `makeVerifyMcpToken` and pass a local JWKS, so they never touch the remote set.

- [ ] **Step 2: Create the client component** `app/oauth/consent/ConsentClient.tsx`:

```tsx
'use client';

import { useEffect, useState } from 'react';
import { useSearchParams } from 'next/navigation';
import { createClient, type SupabaseClient } from '@supabase/supabase-js';

const supabaseUrl = process.env.NEXT_PUBLIC_SUPABASE_URL!;
const supabaseKey = process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!;

type Details = { client?: { name?: string; client_name?: string }; scopes?: string[] };

export default function ConsentClient() {
  const params = useSearchParams();
  const authorizationId = params.get('authorization_id') ?? '';
  const [supabase] = useState<SupabaseClient>(() => createClient(supabaseUrl, supabaseKey));
  const [state, setState] = useState<'loading' | 'login' | 'consent' | 'working' | 'error'>('loading');
  const [details, setDetails] = useState<Details | null>(null);
  const [message, setMessage] = useState('');

  useEffect(() => {
    if (!authorizationId) { setState('error'); setMessage('Missing authorization_id.'); return; }
    (async () => {
      const { data: { session } } = await supabase.auth.getSession();
      if (!session) { setState('login'); return; }
      const { data, error } = await supabase.auth.oauth.getAuthorizationDetails(authorizationId);
      if (error) { setState('error'); setMessage(error.message); return; }
      if (data && 'redirect_url' in data) { window.location.href = data.redirect_url; return; }
      setDetails(data as Details);
      setState('consent');
    })();
  }, [authorizationId, supabase]);

  async function signIn(provider: 'google' | 'github') {
    await supabase.auth.signInWithOAuth({ provider, options: { redirectTo: window.location.href } });
  }

  async function approve() {
    setState('working');
    const { error } = await supabase.auth.oauth.approveAuthorization(authorizationId);
    if (error) { setState('error'); setMessage(error.message); }
  }

  async function deny() {
    setState('working');
    const { error } = await supabase.auth.oauth.denyAuthorization(authorizationId);
    if (error) { setState('error'); setMessage(error.message); }
  }

  const wrap: React.CSSProperties = { fontFamily: 'system-ui', maxWidth: 420, margin: '15vh auto', padding: 24, lineHeight: 1.5 };
  const btn: React.CSSProperties = { display: 'block', width: '100%', padding: '10px 14px', margin: '8px 0', fontSize: 15, cursor: 'pointer' };

  if (state === 'loading' || state === 'working') return <main style={wrap}>Working…</main>;
  if (state === 'error') return <main style={wrap}><h1 style={{ fontSize: 18 }}>Something went wrong</h1><p>{message}</p></main>;

  if (state === 'login') {
    return (
      <main style={wrap}>
```
